## 1.0 บทนำ: ทำไมการแสดงผลข้อมูล (Data Visualization) จึงสำคัญ

ลองจินตนาการว่าผู้วิเคราะห์ได้รับไฟล์ข้อมูลในรูปแบบ CSV ซึ่งประกอบด้วยข้อมูลมากกว่า 50,000 แถว พร้อมคำถามง่าย ๆ เพียงข้อเดียวคือ

> **“เกิดอะไรขึ้นในข้อมูลชุดนี้?”**

การพิจารณาข้อมูลตัวเลขดิบจำนวนมากในรูปแบบตารางไม่ใช่วิธีการที่มีประสิทธิภาพ สมองมนุษย์ไม่สามารถรับรู้รูปแบบ แนวโน้ม หรือความสัมพันธ์ที่ซ่อนอยู่ในข้อมูลจำนวนมหาศาลได้โดยตรง เหตุผลสำคัญคือ มนุษย์เป็น *นักคิดเชิงภาพ* (visual thinkers) โดยธรรมชาติ ในขณะที่ข้อมูลดิบเป็นเพียง “ข้อมูลนามธรรม” (abstract data) ซึ่งไม่ได้ถูกออกแบบมาให้มนุษย์ประมวลผลโดยตรง

การแสดงผลข้อมูล (Data Visualization) จึงมีบทบาทเชิงกลยุทธ์ในการลดภาระทางการรับรู้ (cognitive load) โดยแปลงข้อมูลที่จับต้องไม่ได้ให้กลายเป็น “โครงสร้างที่รับรู้ได้” (perceptible structure) เช่น กราฟและแผนภูมิ สิ่งเหล่านี้ช่วยให้มนุษย์สามารถให้เหตุผล เปรียบเทียบ และตัดสินใจได้อย่างรวดเร็วและแม่นยำยิ่งขึ้น

> **การแสดงผลข้อมูลไม่ใช่การทำให้ข้อมูลดูสวยงาม แต่เป็นการทำให้ข้อมูลสามารถนำมาขบคิดได้ (thinkable)**

## 2.0 กระบวนการเปลี่ยนข้อมูลสู่ความเข้าใจเชิงลึก  
### Data → Information → Insight

ข้อมูลดิบในตัวมันเองยังไม่มีความหมาย จำเป็นต้องผ่านกระบวนการจัดระเบียบ ตีความ และสังเคราะห์ จึงจะสามารถสร้างคุณค่าเชิงความรู้ได้ กระบวนการนี้สามารถอธิบายได้เป็นสามระดับ ได้แก่ Data, Information และ Insight

### ตัวอย่าง: คะแนนสอบของนักเรียน

| Student | Score |
|--------|-------|
| A | 55 |
| B | 78 |
| C | 92 |

ระดับของความเข้าใจสามารถอธิบายได้ดังนี้

- **Data (ข้อมูล)**  
  คะแนน 55, 78, 92 เป็นเพียงตัวเลขดิบที่ยังไม่บ่งบอกความหมายในเชิงภาพรวม

- **Information (ข้อมูลสารสนเทศ)**  
  การจัดกลุ่มหรือแสดงการกระจายตัวของคะแนน เช่น มีนักเรียนได้คะแนนสูงหรือต่ำกี่คน

- **Insight (ความเข้าใจเชิงลึก)**  
  การตีความรูปแบบ เช่น  
  *“นักเรียนส่วนใหญ่สอบผ่าน แต่มีกลุ่มหนึ่งที่มีความเสี่ยงและต้องการความช่วยเหลือเพิ่มเติม”*

> **Insight ไม่ได้มีอยู่โดยตรงในข้อมูล แต่เกิดจากการนำเสนอข้อมูลและการตีความ**

## 3.0 บทบาทของการแสดงผลข้อมูลในกระบวนการวิเคราะห์

### การแสดงผลข้อมูลในฐานะเครื่องมือสำรวจ (Exploratory Tool)

ความเข้าใจผิดที่พบบ่อยคือ การสร้างกราฟเป็นเพียงขั้นตอนสุดท้ายของการนำเสนอผลลัพธ์ แต่ในความเป็นจริง การแสดงผลข้อมูลเป็นหัวใจของขั้นตอน **Exploratory Data Analysis (EDA)**

### ลำดับขั้นตอนการวิเคราะห์ข้อมูล

1. การรวบรวมข้อมูล (Collect data)  
2. การทำความสะอาดข้อมูล (Clean it)  
3. การสำรวจข้อมูลด้วยภาพ (Explore it visually)  
4. การวิเคราะห์เชิงสถิติ (Analyze it statistically)  
5. การสื่อสารผลลัพธ์ (Communicate results)

ขั้นตอนที่ 3 ช่วยให้ตรวจจับความผิดปกติ เห็นโครงสร้างของข้อมูล และกำหนดทิศทางการวิเคราะห์ได้อย่างถูกต้อง

> **หากข้ามขั้นตอนการแสดงผลข้อมูล มักจะวิเคราะห์ปัญหาผิดจุด**

## 4.0 การสาธิตด้วย Python: จากทฤษฎีสู่การปฏิบัติ

### Python ในฐานะสภาพแวดล้อมสำหรับการคิดวิเคราะห์

```python
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

sns.set(style="whitegrid")
````

### การสำรวจข้อมูลเบื้องต้น

```python
df = pd.read_csv("students.csv")

df.head()
df.info()
```

คำสั่ง `.info()` เป็นขั้นตอนที่สำคัญ เพราะการสร้างกราฟโดยไม่เข้าใจชนิดข้อมูลอาจนำไปสู่การตีความที่ผิดพลาด

---

## 4.1 การเลือกกราฟตามชนิดของข้อมูล

### ข้อมูลเชิงหมวดหมู่ (Categorical Data)

```python
df['gender'].value_counts().plot(kind='bar')
plt.title("Gender Distribution")
plt.show()
```

กราฟนี้เหมาะสำหรับการเปรียบเทียบปริมาณในแต่ละกลุ่ม และช่วยตั้งสมมติฐานเบื้องต้น

---

### ข้อมูลเชิงปริมาณ (Numerical Data)

```python
plt.hist(df['score'], bins=10)
plt.title("Score Distribution")
plt.show()
```

สิ่งที่ควรพิจารณา ได้แก่

* แนวโน้มเข้าสู่ส่วนกลาง
* การกระจายตัว
* ค่าผิดปกติ (outliers)

---

### การสำรวจความสัมพันธ์ (Scatter Plot)

```python
sns.scatterplot(x='study_hours', y='score', data=df)
plt.title("Study Hours vs Score")
plt.show()
```

> กราฟนี้ไม่สามารถพิสูจน์เหตุและผล (causation)
> แต่ช่วยสร้างสมมติฐานสำหรับการวิเคราะห์เชิงสถิติ

**Visualization สร้างสมมติฐาน
Analytics ทดสอบสมมติฐาน**

## 4.2 การคิดเชิงวิพากษ์ต่อกราฟ

ก่อนเชื่อกราฟใด ๆ ควรถามคำถามต่อไปนี้

1. แกน Y เริ่มที่ศูนย์หรือไม่
2. มีการใช้กราฟสามมิติที่บิดเบือนการรับรู้หรือไม่
3. สีที่ใช้มีความหมายหรือไม่
4. กราฟสื่อสารประเด็นหลักได้ชัดเจนหรือไม่
5. มีส่วนใดที่อาจทำให้เข้าใจผิดหรือไม่
